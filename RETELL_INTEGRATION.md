# Retell Agent Integration Guide

## Overview

This guide explains how to take the **Retell Agent Spec JSON** generated by the Clara AI pipeline and configure it in Retell.

## Getting Started with Retell

### 1. Create a Retell Account

1. Visit [https://retellai.com](https://retellai.com)
2. Sign up for a **free account** (no credit card required for free tier)
3. Verify your email
4. Log in to your Retell dashboard

### 2. Get Your API Credentials (if using API)

If Retell offers free-tier API access:

1. Go to **Settings** → **API Keys**
2. Create a new API key for development
3. Copy the API key (keep it private)
4. Store in a `.env` file:

```bash
RETELL_API_KEY=your_api_key_here
```

**Note:** If Retell does not allow free-tier programmatic agent creation, follow the **Manual Import** method below.

---

## Method A: Programmatic API Integration (if available)

### 1. Create Agent via API

Use the generated Retell Agent Spec JSON:

```bash
curl -X POST https://api.retellai.com/agents \
  -H "Authorization: Bearer $RETELL_API_KEY" \
  -H "Content-Type: application/json" \
  -d @outputs/accounts/account_01/v1/retell_agent_spec.json
```

### 2. Response Example

```json
{
  "agent_id": "agent_abc123xyz",
  "agent_name": "Acme Fire Protection AI Answering Agent",
  "status": "active",
  "system_prompt": "You are the AI voice receptionist...",
  "created_at": "2026-03-06T16:40:00Z"
}
```

### 3. Test Your Agent

Use the agent ID to:

1. Make test calls via Retell web interface
2. Get call transcripts and analytics
3. Iterate and update via PATCH `/agents/{agent_id}`

## Method B: Manual Import (Recommended for Free Tier)

If Retell's free tier does not support API agent creation:

### 1. Open Your Generated Agent Spec

From your outputs:

```bash
cat outputs/accounts/account_01/v1/retell_agent_spec.json
```

Example output:

```json
{
  "agent_name": "Acme Fire Protection AI Answering Agent",
  "system_prompt": "You are the AI voice receptionist for Acme Fire Protection...",
  "voice_style": "professional",
  "key_variables": {
    "timezone": "Eastern",
    "business_hours": "Monday, Friday 9 am-5 pm Eastern",
    "office_address": "123 Main Street.",
    "emergency_routing": "our dispatch"
  },
  "call_transfer_protocol": "Transfer per routing rules...",
  "fallback_protocol_if_transfer_fails": "If transfer fails..."
}
```

### 2. Manual Steps in Retell UI

1. Log in to Retell dashboard
2. Click **+ Create New Agent**
3. Fill in:
   - **Agent Name:** Copy from `agent_name` field
   - **Voice Style:** Copy from `voice_style` field (e.g., "professional")
4. In **System Prompt** section:
   - Clear default text
   - Paste the entire `system_prompt` string from the JSON
5. In **Key Variables** section:
   - Add each variable from the JSON:
     - Key: `timezone` → Value: `Eastern`
     - Key: `business_hours` → Value: `Monday, Friday 9 am-5 pm Eastern`
     - Key: `office_address` → Value: `123 Main Street.`
     - Key: `emergency_routing` → Value: `our dispatch`
     - (Add any others from `key_variables`)
6. In **Transfer Protocol** section:
   - Paste `call_transfer_protocol` value
7. In **Fallback Protocol** section:
   - Paste `fallback_protocol_if_transfer_fails` value
8. Click **Save & Test**

### 3. Update with v2 Spec (After Onboarding)

When you run onboarding updates:

```bash
cat outputs/accounts/account_01/v2/retell_agent_spec.json
```

1. Go back to the agent in Retell
2. Click **Edit**
3. Replace the **System Prompt** with the new v2 prompt
4. Update **Key Variables** (e.g., business hours may have changed)
5. Update transfer and fallback protocols if needed
6. Click **Save & Test**

## Understanding the Generated Spec

### Account Memo JSON

Contains structured data extracted from transcripts:

```json
{
  "account_id": "account_01",
  "company_name": "Acme Fire Protection",
  "business_hours": {
    "days": ["Monday", "Friday"],
    "start": "9 am",
    "end": "5 pm",
    "timezone": "Eastern"
  },
  "office_address": "123 Main Street.",
  "services_supported": ["sprinkler", "fire alarm", "inspection"],
  "emergency_definition": ["fire", "alarm", "emergency"],
  "emergency_routing_rules": ["our dispatch"],
  "call_transfer_rules": {
    "timeouts": "60 seconds",
    "retries": "2",
    "if_fails_say": "If transfer fails after 60 seconds, dispatch must be notified."
  },
  "integration_constraints": ["Never create sprinkler jobs in ServiceTrade from the agent"],
  "questions_or_unknowns": []
}
```

### Retell Agent Spec JSON

Combines the memo into a ready-to-use agent configuration:

```json
{
  "version": "v1",
  "agent_name": "Acme Fire Protection AI Answering Agent",
  "voice_style": "professional",
  "system_prompt": "You are the AI voice receptionist...",
  "key_variables": {
    "timezone": "Eastern",
    "business_hours": "Monday, Friday 9 am-5 pm Eastern",
    "office_address": "123 Main Street.",
    "emergency_routing": "our dispatch",
    "caller_name": "",
    "caller_phone": "",
    "call_reason": ""
  },
  "call_transfer_protocol": "Transfer per routing rules; respect timeout and retries...",
  "fallback_protocol_if_transfer_fails": "If transfer fails after 60 seconds..."
}
```

## Versioning: v1 → v2

### What Changes?

When onboarding data arrives, the pipeline generates **v2** updates:

```bash
outputs/accounts/account_01/v1/   # Initial from demo
outputs/accounts/account_01/v2/   # Updated from onboarding

changelog/account_01_changelog.json  # What changed and why
changelog/account_01_changelog.md    # Human-readable diff
```

### Example Changelog

```markdown
# Changelog: account_01
**v1 → v2**

9 memo field(s) changed, 7 spec field(s) changed.

## Account memo changes (field-level diff)

**Field changed:** business_hours.timezone
- Old: Eastern
- New: EST

**Field changed:** office_address
- Old: 123 Main Street.
- New: 123 Main Street, Suite 100.

**Field changed:** call_transfer_rules.timeouts
- Old: Not specified
- New: 60 seconds
```

### To Update Your Agent in Retell:

1. Review the changelog to see what changed
2. Open your agent in Retell
3. Update the fields that changed (usually System Prompt and Key Variables)
4. Test the updated agent
5. Publish when satisfied

## Troubleshooting

### "System prompt is too long"
- Retell may have character limits
- Trim the prompt or remove verbose explanations
- Focus on the essential flows and routing logic

### "Invalid JSON in key variables"
- Ensure all values are strings (not objects or arrays)
- Use plain text for variable values
- Avoid nested JSON in values

### "Agent not responding"
- Check that the system prompt includes proper greeting and flow
- Verify that business hours and emergency routing are clearly specified
- Test with simple calls first (e.g., "Hello, what is your business hours?")

### "Can't import programmatically"
- Fall back to **Manual Import** method
- Retell's free tier may limit API access
- This is normal and expected for zero-cost solutions
